# Códigos de Projetos de Engenharia de Dados (Python, SQL, Spark) — omarcocaja

Este repositório reúne os **artefatos desenvolvidos ao vivo nas aulas da comunidade**: códigos, notebooks, anotações, checklists e materiais de apoio criados durante as lives.

As lives são focadas em **Engenharia de Dados na prática**, com aplicações reais de **Python**, **SQL**, **Spark/PySpark**, pipelines e padrões que ajudam você a sair do “consumo de conteúdo” e começar a **produzir entregas reproduzíveis**.

Esse repositório é público, para que você possa praticar no Databricks Free Edition ou qualquer outro lugar que seja mais confortável para você!

---

## O que tem aqui

Em cada live, você encontra uma pasta no formato:

- `LIVE - AAAAMMDD`

Dentro de cada pasta ficam os **artefatos criados naquela aula** (ex.: notebooks `.ipynb`, scripts `.py`, arquivos `.sql`, checklists `.md`, imagens, anotações, etc.).

Exemplos de lives já presentes:
- `LIVE - 20251013`
- `LIVE - 20251124`
- `LIVE - 20260302`

> Dica: use a busca do GitHub para encontrar termos como **pyspark**, **sql**, **pipeline**, **bronze**, **silver**, **gold**, **dag**, **airflow**, **data quality**, etc.

---

## Como navegar pelas aulas

1. Abra a pasta da live (ex.: `LIVE - 20260302`)
2. Leia os arquivos da aula (geralmente `README`, `.md` ou o próprio notebook)
3. Rode os códigos localmente (ou no Databricks)
4. Reaplique a lógica em um dataset seu (é aqui que o aprendizado “fixa”)

---

## Requisitos (varia por live)

Como cada live pode ter ferramentas diferentes, os requisitos podem mudar. No geral, você vai ver uma combinação de:

- **Python 3.10+**
- **Jupyter Notebook / VS Code**
- **Pandas**
- **SQL** (SQLite/Postgres/BigQuery — depende do tema)
- **Spark/PySpark** (quando a aula for sobre processamento distribuído)

Se a live exigir algo específico (ex.: Docker, Spark local, Postgres, Databricks), isso deve estar documentado **dentro da pasta daquela live**.
